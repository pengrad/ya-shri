2. Опишите своими словами, что происходит в браузерах при клике на ссылку yandex.ru.

1. Обрабатывается событие onclick, вызывается соответствующий JS-код.
2. Создается и открывается некий HttpConnection по указанному URL.
3. Осуществляется Http-запрос, в качестве параметров используется заголовки запроса (Request Headers).
Такие как: Method, Accept, User-Agent. Передаются куки для этого сайта.
4. Открывается входной поток, и считываются статус и заголовки ответа (Response Headers).
Content-Length, Content-Type.
5. Если контент есть (Content-Length > 0), смотрится Content-Type и, если браузер не может его отобразить своими средствами, после диалога с пользователем происходит загрузка в файл.
Если браузер может его отобразить (предположим это text/html):
6. Html загружается и по мере загрузки парсится.
7. Происходит рендеринг загруженного контента и параллельно с ним загрузка остальной части страницы, а также других ресурсов, подключенных к странице.
8. Загрузка ресурсов (css, js, images) осуществляется с передачей Request Headers и с получением Response Headers.
В случае получения статуса "304 Not Modified", ресурс может считаться из кэша.
Загрузка происходит параллельно в несколько потоков. Число потоков ограниченно по-разному в разных браузерах.
9. В ходе загрузки при интерпретации JS или CSS может произойти перерендеринг уже отображенных частей.
10. После загрузки на диск сохраняются ресурсы и куки.

    switch (price.charAt(0)) {
        case '€':
            return 'R' + (40 * price.slice(1));

        case '¥':
            return 'R' + (41 * price.slice(1));
    }
}
